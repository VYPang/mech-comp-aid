## 1. Prerequisites & Installation

This project utilizes `uv` for strict environment and dependency resolution. The core stack includes `numpy` for C-level matrix vectorization, `numpy-stl` for STL mesh loading, `scikit-image` for Marching Cubes isosurface extraction, and `pyvista` for GPU-accelerated 3D plotting. Additionally, `typer` and `rich` are used to provide the CLI.

Make sure your `uv` environment is synchronized by exactly replicating the environment using the generated `uv.lock` file:

```bash
# Installs all required dependencies defined in uv.lock into a local virtual environment (.venv)
uv sync
```

### Alternatively (Without `uv` project tracking)
If you just want to install the packages into your current environment directly without relying on `uv.lock`:
```bash
uv pip install numpy numpy-stl pyvista scikit-image typer rich typing-extensions
```
Or with standard pip:
```bash
pip install numpy numpy-stl pyvista scikit-image typer rich typing-extensions
```

## 2. Exercise 1-1: Visualization, Rotation and Area

The `visSTL.py` script makes use of `typer` and `rich` to provide an interactive command-line interface for viewing and rotating 3D STL files. 

### Available Commands

There are three main commands available in this script: `visualize`, `rotate`, and `area`. To use the tool, specify the sub-command and the optional flags.

#### A. `visualize`

This command lets you view the original 3D models. You can view the `twisted` vase, the `tessa` vase, or `both` side by side.

**Options:**
* `--model twisted`: View only the Twisted Vase.
* `--model tessa`: View only the Tessa Vase.
* `--model both`: (Default) View both vases side-by-side.

**Usage Examples:**
```bash
# View both models side-by-side (default)
python project1/exercise1-1/visSTL.py visualize

# View only the Twisted Vase
python project1/exercise1-1/visSTL.py visualize --model twisted
```

#### B. `rotate`

This command rotates the selected 3D model (π/4 clock-wise about the x-axis, then π/3 counter-clock-wise about the y-axis) and visualizes the results. It displays three plots side-by-side for comparison:
1. The original model.
2. The model rotated using custom rotation matrices.
3. The model rotated using `numpy-stl` built-in rotation functions.

**Options:**
* `--model twisted`: (Default) Rotate the Twisted Vase.
* `--model tessa`: Rotate the Tessa Vase.

**Usage Examples:**
```bash
# Rotate and compare the Twisted Vase (default)
python project1/exercise1-1/visSTL.py rotate

# Rotate and compare the Tessa Vase
python project1/exercise1-1/visSTL.py rotate --model tessa
```

#### C. `area`

This command calculates the total surface area of the chosen 3D model. It calculates the area using two separate methods and outputs both for comparison:
1. Manually computing the cross-product magnitude of each constituent triangular face and summing them together. 
2. Invoking the built-in properties provided by the `numpy-stl` package.

**Options:**
* `--model twisted`: (Default) Calculate the area of the Twisted Vase.
* `--model tessa`: Calculate the area of the Tessa Vase.

**Usage Examples:**
```bash
# Calculate the surface area of the Twisted Vase (default)
python project1/exercise1-1/visSTL.py area

# Calculate the surface area of the Tessa Vase
python project1/exercise1-1/visSTL.py area --model tessa
```

You can always consult the built-in help documentation by passing the `--help` flag:
```bash
python project1/exercise1-1/visSTL.py --help
python project1/exercise1-1/visSTL.py visualize --help
```

## 3. Exercise 1-2: Program Periodic Surfaces

This module provides a mathematical implementation to compute and visualize **Triply Periodic Minimal Surfaces (TPMS)**. These continuous, non-intersecting surfaces are defined by implicit equations and commonly used in mechanical meterials, bone scaffolds, and generative lattice designs.

The currently supported mathematically approximated surface typologies include:
1. **P** - Primitive
2. **D** - Diamond
3. **G** - Gyroid

### Overview of Mathematical Formulation

The surfaces are generated by evaluating the general implicit periodic function $\psi(\mathbf{r}) = 0$:

$$ \psi(\mathbf{r}) = \sum^L_{l=1}\sum^M_{m=1} \mu_{m} \cos(2\pi \kappa (\mathbf{p}_m^T \cdot \mathbf{r})) = 0 $$

Where:
*   $\mathbf{r}$ is the location vector computed homogeneously as $[x, y, z, 1]$.
*   $\mu_m$ is the sequence of periodic moment weights.
*   $\mathbf{p}_m$ are standard basis matrix vectors mapped statically for `P`, `D`, and `G`.
*   $\kappa$ (Kappa) acts as a global scale multiplier to compress or stretch structural frequencies.

### CLI Usage

A command-line interface (powered by `typer` and formatted with `rich`) allows fast configuration, mathematical verification, and 3D rendering. Run `python project1/exercise1-2/periodic_surfaces.py --help` to outline available arguments.

#### A. Evaluate a Specific Point

Use the `evaluate` command to print the exact sum/computation for a static $X, Y, Z$ coordinate in 3D cartesian space.

**Syntax:**
```bash
python project1/exercise1-2/periodic_surfaces.py evaluate [SURFACE_TYPE] [OPTIONS]
```

**Examples:**
```bash
# Evaluate the Primitive surface at the origin point
python project1/exercise1-2/periodic_surfaces.py evaluate P --x 0 --y 0 --z 0

# Evaluate the Gyroid surface at specific coordinates with a kappa scale of 0.5
python project1/exercise1-2/periodic_surfaces.py evaluate G -x 0.5 -y 1.0 -z 1.5 -k 0.5
```

#### B. Plot the 3D Surface

Use the `plot` command. This casts a structured matrix across a bounding box, evaluates the implicit fields across thousands of vertices leveraging Numpy vectorization, mathematically extracts a surface manifold via `measure.marching_cubes`, and pushes the triangulated result into a `pyvista` interactive display.

**Syntax:**
```bash
python project1/exercise1-2/periodic_surfaces.py plot [SURFACE_TYPE] [OPTIONS]
```

**Options:**
*   `--kappa` bounds scale multiplier (default: `1.0`)
*   `--min` overrides minimum grid bound along all X/Y/Z axes (default: `-4.0`).
*   `--max` overrides maximum grid bound along all X/Y/Z axes (default: `4.0`).
*   `--res` density of uniform plotting grid; impacts the visual smoothness and terminal runtime heavily (default: `50`).

**Examples:**
```bash
# Plot a standard Diamond surface structure
python project1/exercise1-2/periodic_surfaces.py plot D

# Plot a concentrated Gyroid surface block with finer resolution (80x80x80 bounded between [-2.0, 2.0])
python project1/exercise1-2/periodic_surfaces.py plot G --min -2.0 --max 2.0 --res 80
```

## 4. Exercise 1-3: Implicit Surfaces and Metamorphosis

This module provides tools to construct, combine, and morph implicit 3D geometries. It uses scalar fields ($F(x,y,z) = 0$) and extracts polygonal meshes using the Marching Cubes algorithm for visualization via PyVista. The CLI provides commands corresponding to three primary tasks.

### CLI Usage

A `typer`-based CLI structure is provided to easily visualize the problems. Run `python project1/exercise1-3/implicit.py --help` for available commands.

#### A. Vizualise a Sphere (Problem 1)

Visualizes an implicitly defined sphere centered at $(0,0,0)$ with a radius of $1$.

**Syntax & Example:**
```bash
python project1/exercise1-3/implicit.py sphere
```

#### B. Boolean Union (Problem 2)

Visualizes the boolean union of two implicit geometries: a $1 \times 1 \times 1$ box and a cylinder aligned along the Z-axis. You can compute this union using standard Min/Max operations or an R-function.

**Options:**
*   `--method`: Compute using `minmax` (standard boolean logic) or `rfunc` (smooth representation blending via R-function approximation). Default is `minmax`.
*   `--alpha`: Adjust the blending smoothness parameter when using R-functions. Default is `0.5`.

**Examples:**
```bash
# Compute union using min/max logic
python project1/exercise1-3/implicit.py union
# or
python project1/exercise1-3/implicit.py union --method minmax

# Compute union using R-functions
python project1/exercise1-3/implicit.py union --method rfunc

# Compute union using R-functions with a custom alpha blending metric
python project1/exercise1-3/implicit.py union --method rfunc --alpha 0.8
```

#### C. Metamorphosis (Problem 3)

Generates and visualizes a sequence of blending transformations (metamorphosis) between a sphere and the box-cylinder union over 11 steps ($\mu \in [0, 1]$). Each step is visualized sequentially; simply close the window to advance to the next step.

**Syntax & Example:**
```bash
python project1/exercise1-3/implicit.py metamorphose
```
